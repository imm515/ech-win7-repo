# 快速开始指南

## 📦 步骤 1：准备 Cloudflare Worker

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 进入 **Workers & Pages**
3. 创建新的 Worker
4. 复制 `_worker.js` 的内容到编辑器
5. 修改第 3 行：
   ```javascript
   const CF_FALLBACK_IPS = ['your-proxy-ip.com'];  // 填入优选 IP
   ```
6. （可选）修改第 13 行设置 token：
   ```javascript
   const token = 'your-secret-token';  // 自定义 token
   ```
7. 点击 **Deploy** 部署
8. 记下 Worker 域名，例如：`your-worker.workers.dev`

---

## 🚀 步骤 2：推送代码到 GitHub

```bash
# 初始化 Git 仓库（如果是第一次）
git init
git branch -M main

# 添加远程仓库
git remote add origin https://github.com/你的用户名/你的仓库名.git

# 提交所有文件
git add .
git commit -m "Win7 CDN优选客户端"

# 推送到 GitHub
git push -u origin main
```

---

## 🔨 步骤 3：手动触发 GitHub Actions 编译

1. 访问你的 GitHub 仓库
2. 点击顶部标签栏的 **Actions**
3. 在左侧列表中找到 **"Auto Build Win7 x64 & Sync Deps"**
4. 点击进入工作流详情页
5. 点击右侧的 **"Run workflow"** 按钮
6. 选择分支（通常是 `main`），点击绿色的 **"Run workflow"**
7. 等待编译完成（约 1-2 分钟）

---

## 📥 步骤 4：下载编译好的程序

1. 在 Actions 页面，等待任务状态变为 ✅（绿色对勾）
2. 点击进入该任务详情
3. 滚动到底部的 **Artifacts** 区域
4. 下载 **proxy-win7-x64**

---

## ⚙️ 步骤 5：配置启动脚本

1. 解压下载的压缩包
2. 右键编辑 `start.bat`
3. 修改以下参数：

```batch
REM 本地监听地址（默认即可）
set LISTEN_ADDR=127.0.0.1:1080

REM Cloudflare Worker 地址（必填！）
set SERVER_ADDR=your-worker.workers.dev:443

REM 可选：服务器 IP（域名解析失败时使用），不填则不传递该参数
set SERVER_IP=

REM 可选：认证令牌（与 Worker 中的 token 一致），不填则不传递该参数
set TOKEN=
```

4. 保存文件

---

## ▶️ 步骤 6：运行CDN优选

1. 双击 `start.bat`
2. 看到 `[启动] 正在启动CDN优选服务器...` 表示成功
3. 窗口保持打开状态
4. 按 `Ctrl+C` 停止CDN优选

---

## 🌐 步骤 7：配置浏览器CDN优选

### Firefox
1. 打开 **设置** → **网络设置**
2. 选择 **手动配置CDN优选**
3. 填写：
   - SOCKS CDN优选：`127.0.0.1` 端口：`1080`
   - SOCKS v5
4. 勾选 **远程 DNS**

### Chrome/Edge
1. 安装CDN优选插件（如 SwitchyOmega）
2. 配置：
   - CDN优选协议：SOCKS5
   - 服务器：`127.0.0.1`
   - 端口：`1080`

## ❓ 常见问题

### Q: Actions 编译失败？

**A:** 检查 go.mod 文件是否存在，内容是否正确：
```go
module proxy-client

go 1.20

require github.com/gorilla/websocket v1.5.1
```

### Q: 连接 Worker 失败？

**A:**
1. 检查 Worker 域名是否正确
2. 尝试添加 `SERVER_IP` 参数
3. 检查防火墙是否阻止 443 端口

### Q: 浏览器无法上网？

**A:**
1. 检查CDN优选配置是否正确
2. 确认 start.bat 正在运行
3. 查看浏览器控制台是否有错误信息
